# Лабораторна робота №3  
## Дослідження обмежень ресурсів у середовищі Docker

### Мета роботи

Ознайомитися з механізмом обмеження ресурсів у Linux за допомогою команди `ulimit`, а також дослідити як ці обмеження впливають на виконання програм.

# Підготовка середовища
Для виконання завдань використовувався Docker контейнер з образом Ubuntu.

Запуск контейнера відбувається за допомогою команди:

///
docker run -it --rm --privileged -v "$PWD":/work -w /work ubuntu:24.04 bash
///

Пояснення параметрів:
- `-it` — запуск контейнера у інтерактивному режимі  
- `--rm` — контейнер видаляється після завершення  
- `-v "$PWD":/work` — підключає поточну папку до контейнера  
- `-w /work` — встановлює робочу директорію

Це дозволяє працювати з файлами контейнера, але зберігати їх у файловій системі Ubuntu.

---

# Встановлення необхідних програм

Після запуску контейнера потрібно встановити необхідні інструменти.
///
apt update
apt install gcc nano
///

Було встановлено:
- **gcc** — для компіляції програм на C  
- **nano** — текстовий редактор для створення та редагування файлів  
---

# Проблеми під час підготовки

Під час запуску Docker виникла помилка доступу:
permission denied while trying to connect to the Docker daemon
Ця проблема виникає через відсутність прав доступу до Docker.

Рішення:
///
sudo docker run -it --rm --privileged -v "$PWD":/work -w /work ubuntu:24.04 bash
///
Після цього контейнер запустився нормально.

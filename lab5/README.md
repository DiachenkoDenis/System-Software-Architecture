# Практична робота №5 
## Помилки роботи з пам’яттю в Linux та засоби їх усунення

У цій лабораторній роботі досліджуються типові помилки при роботі з пам’яттю в програмах на мові C у середовищі Linux. Такі помилки можуть призводити до аварійного завершення програми, пошкодження даних або витоків пам’яті.

Особливу увагу приділено фрагментації пам’яті - ситуації, коли сумарно вільної пам’яті достатньо, але через її розбиття на окремі фрагменти неможливо виділити один великий безперервний блок.

## Завдання 5.6 - Побудувати приклад фрагментації пам’яті

У цьому завданні необхідно створити приклад фрагментації пам’яті, при якому велика алокація пам’яті завершується невдачею, хоча сумарно вільної пам’яті достатньо.

Фрагментація виникає тоді, коли після декількох операцій виділення та звільнення пам’яті вільні блоки розміщуються у різних частинах пам’яті і не утворюють один великий безперервний сегмент.

У такій ситуації система не може виділити великий блок пам’яті, навіть якщо загальний обсяг вільної пам’яті достатній.

Для демонстрації цього явища була створена проста модель пам’яті фіксованого розміру, у якій реалізовано власний механізм алокації та звільнення блоків.

## Код програми:
```c
#include <stdio.h>

#define ARENA_SIZE 64

static int used[ARENA_SIZE];

int alloc_block(int size) {
    int count = 0;
    int start = -1;

    for (int i = 0; i < ARENA_SIZE; i++) {
        if (used[i] == 0) {
            if (count == 0) {
                start = i;
            }
            count++;

            if (count == size) {
                for (int j = start; j < start + size; j++) {
                    used[j] = 1;
                }
                return start;
            }
        } else {
            count = 0;
            start = -1;
        }
    }

    return -1;
}

void free_block(int start, int size) {
    for (int i = start; i < start + size; i++) {
        used[i] = 0;
    }
}

void print_arena() {
    for (int i = 0; i < ARENA_SIZE; i++) {
        printf("%c", used[i] ? '#' : '.');
    }
    printf("\n");
}

int count_free_memory() {
    int total = 0;

    for (int i = 0; i < ARENA_SIZE; i++) {
        if (used[i] == 0) {
            total++;
        }
    }

    return total;
}

int main() {

    int a = alloc_block(8);
    int b = alloc_block(16);
    int c = alloc_block(8);
    int d = alloc_block(16);
    int e = alloc_block(8);

    printf("Initial allocation:\n");
    print_arena();

    free_block(b, 16);
    free_block(d, 16);

    printf("After freeing two middle blocks:\n");
    print_arena();

    printf("Total free memory: %d bytes\n", count_free_memory());

    int big = alloc_block(24);

    if (big == -1) {
        printf("Large allocation failed\n");
    } else {
        printf("Large allocation succeeded at %d\n", big);
    }

    return 0;
}
```

## Результати виконання

Після компіляції та запуску програми було отримано наступний результат:
```
Initial allocation:
########################################################........

After freeing two middle blocks:
########................########................########........

Total free memory: 40 bytes
Large allocation failed
```
Спочатку програма виконує кілька послідовних алокацій пам’яті у фіксованій області розміром 64 байти. У результаті більша частина пам’яті стає зайнятою.

Після цього два середні блоки пам’яті звільняються. У пам’яті утворюються дві окремі вільні області. Сумарний обсяг вільної пам’яті становить 40 байт.

Однак при спробі виділити новий блок пам’яті розміром 24 байти алокація завершується невдачею. Це відбувається через те, що в пам’яті немає одного безперервного вільного блоку достатнього розміру, хоча сумарно вільної пам’яті вистачає.

## Висновок

У ході виконання завдання було продемонстровано явище фрагментації пам’яті.

Було встановлено, що навіть якщо сумарно вільної пам’яті достатньо, велика алокація може завершитися невдачею через відсутність одного безперервного вільного блоку необхідного розміру.

Отриманий результат демонструє принцип роботи фрагментації пам’яті та її вплив на можливість виділення великих блоків пам’яті.

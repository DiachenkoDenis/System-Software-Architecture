## Дослідження обмежень виділення пам'яті у Linux

У даній практичній роботі досліджуються обмеження виділення оперативної пам’яті у середовищі Linux за допомогою функції malloc() мови програмування C.

Особлива увага приділяється максимальному обсягу пам’яті, який може бути виділений за один виклик функції malloc(), а також впливу архітектури системи на це обмеження.

## Мета роботи 

Дослідити максимальний обсяг пам’яті, який може бути переданий функції malloc() за один виклик, та зрозуміти обмеження, що виникають через тип даних size_t і архітектуру операційної системи.

## Завдання 4.1 - Дослідження максимального обсягу пам’яті для malloc()

Функція malloc() використовується у мові програмування C для динамічного виділення пам’яті.

Прототип функції має вигляд:
```
void* malloc(size_t size);
```
Параметр size визначає кількість байтів пам’яті, яку потрібно виділити.

Тип даних size_t є беззнаковим цілим типом, який використовується для зберігання розміру об’єктів у пам’яті.

У 64-бітних системах Linux тип size_t має розмір 8 байтів, тобто 64 біти. Теоретично це дозволяє передати у функцію malloc() значення:

2^64 байтів

що приблизно дорівнює 16 ексабайтам пам’яті.

Однак у реальних умовах максимальний обсяг пам’яті, який може бути виділений за один виклик malloc(), становить приблизно:

2^63 байтів

тобто приблизно 8 ексабайт.

Це пояснюється тим, що частина бітів використовується внутрішніми механізмами управління пам’яттю, а також існують обмеження адресного простору процесу.

Для перевірки максимального значення типу size_t була створена тестова програма мовою C.

## Код програми:
```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    size_t max_size = (size_t)-1;
    printf("Maximum value of size_t: %zu\n", max_size);
    void *ptr = malloc(max_size);
    if (ptr == NULL) {
        printf("Memory allocation failed\n");
    } else {
        printf("Memory allocated\n");
        free(ptr);
    }
    return 0;
}
```

## Завдання 4.2 - Дослідження поведінки malloc() при передачі від’ємного аргументу

У даному завданні досліджується, що відбувається при передачі у функцію malloc() від’ємного аргументу, а також як впливає переповнення знакової цілочисельної змінної на результат виділення пам’яті.

Функція malloc() приймає аргумент типу size_t, який є беззнаковим цілим типом даних.
Це означає, що від’ємні значення напряму не можуть бути коректно інтерпретовані як розмір пам’яті.

Якщо у malloc() передати від’ємне значення, воно автоматично перетворюється до типу size_t, тобто у дуже велике додатне число. У результаті функція намагається виділити надто великий обсяг пам’яті.

Для перевірки було створено тестовий приклад, у якому кількість байтів для виділення пам’яті обчислюється за формулою:
```
num = xa * xb;
```

При цьому змінна num оголошується як знаковий тип int.
Якщо результат множення перевищує максимально допустиме значення для типу int, виникає переповнення.

У такому випадку змінна num отримує некоректне значення. На практиці це може бути від’ємне число. Після передачі цього значення у malloc() воно перетворюється до size_t, і функція виконує спробу виділити дуже великий обсяг пам’яті.

У більшості випадків таке виділення пам’яті не виконується, і функція malloc() повертає NULL.

## Код програми:
```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int xa = 50000;
    int xb = 50000;
    int num = xa * xb;
    printf("xa = %d\n", xa);
    printf("xb = %d\n", xb);
    printf("num = xa * xb = %d\n", num);
    void *ptr = malloc(num);

    if (ptr == NULL) {
        printf("malloc failed\n");
    } else {
        printf("malloc succeeded\n");
        free(ptr);
    }

    return 0;
}
```

У даному випадку як на x86_64, так і на x86 функція malloc() повернула NULL, оскільки виділити такий обсяг пам’яті неможливо.

## Завдання 4.3 - Дослідження поведінки malloc(0)

У даному завданні досліджується поведінка функції malloc() при виклику з аргументом 0.

Функція malloc() використовується для динамічного виділення пам’яті та приймає один аргумент - кількість байтів, які необхідно виділити.

Виклик malloc(0) є особливим випадком. Згідно з реалізацією бібліотеки стандартних функцій, така операція може мати два допустимі результати:

функція може повернути NULL

функція може повернути вказівник, що не дорівнює NULL, але який можна лише передати у free()

Тобто, якщо malloc(0) повертає не-NULL вказівник, це ще не означає, що було реально виділено область пам’яті, придатну для використання. Такий вказівник не слід розіменовувати або використовувати для запису даних. Допускається лише його подальша передача у free().

Для перевірки було створено тестову програму, яка викликає malloc(0), перевіряє значення поверненого вказівника та, у випадку якщо він не дорівнює NULL, викликає free().

## Код програми:
```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    void *ptr = malloc(0);
    if (ptr == NULL) {
        printf("malloc(0) returned NULL\n");
    } else {
        printf("malloc(0) returned non-NULL pointer: %p\n", ptr);
        free(ptr);
        printf("Pointer was passed to free()\n");
    }
    return 0;
}
```

## Завдання 4.4 - Аналіз помилки використання malloc() та free()

Розглянемо наступний фрагмент коду:
```c
void *ptr = NULL;
while (<some-condition-is-true>) {
    if (!ptr)
        ptr = malloc(n);
    [... використання ptr ...]
    free(ptr);
}
```
На перший погляд код виглядає коректним, однак у ньому присутня логічна помилка.

Після виклику free(ptr) значення змінної ptr не змінюється і продовжує містити стару адресу пам’яті. У наступній ітерації циклу умова if (!ptr) буде хибною, оскільки ptr не дорівнює NULL. У результаті новий виклик malloc() не відбудеться, але програма може продовжити використовувати вже звільнену пам’ять.

Така ситуація називається use-after-free - використання пам’яті після її звільнення. Це може призводити до непередбачуваної поведінки програми, помилок або аварійного завершення.

## Тестовий приклад з помилкою

## Код програми:
```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr = NULL;
    for (int i = 0; i < 3; i++) {
        if (!ptr)
            ptr = malloc(sizeof(int));
        *ptr = i;
        printf("Iteration %d value = %d\n", i, *ptr);
        free(ptr);
    }
    return 0;
}
```
У цьому прикладі після виклику free(ptr) змінна ptr продовжує містити стару адресу. У наступній ітерації умова if (!ptr) не виконується, і програма намагається використовувати вже звільнену пам’ять.

## Правильний варіант коду

Щоб уникнути цієї проблеми, після виклику free() необхідно встановлювати вказівник у NULL.
```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr = NULL;
    for (int i = 0; i < 3; i++) {
        if (!ptr)
            ptr = malloc(sizeof(int));
        *ptr = i;
        printf("Iteration %d value = %d\n", i, *ptr);
        free(ptr);
        ptr = NULL;
    }
    return 0;
}
```
У цьому випадку після звільнення пам’яті вказівник обнуляється, тому на наступній ітерації умова if (!ptr) виконується, і виклик malloc() відбувається знову.

## Висновок

Правильним підходом є встановлення вказівника у NULL після звільнення пам’яті, що дозволяє уникнути подібних помилок.

## Завдання 4.5 - Поведінка realloc() у випадку неможливості виділення пам’яті

Функція realloc() використовується для зміни розміру вже виділеної області пам’яті.

Прототип функції:
```
void *realloc(void *ptr, size_t size);
```
Параметр ptr є вказівником на раніше виділену область пам’яті, а параметр size визначає новий розмір цієї області.

Якщо realloc() може змінити розмір пам’яті, вона повертає вказівник на новий або той самий блок пам’яті.
Якщо ж система не може виділити необхідний обсяг пам’яті, функція повертає NULL.

При цьому важливо, що початковий блок пам’яті не звільняється, і його необхідно звільнити вручну за допомогою free().

## Код програми:
```
#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr = malloc(10 * sizeof(int));
    if (!ptr) {
        printf("Initial malloc failed\n");
        return 1;
    }
    printf("Initial allocation successful\n");
    size_t huge_size = (size_t)1024 * 1024 * 1024 * 1024;
    int *new_ptr = realloc(ptr, huge_size);
    if (!new_ptr) {
        printf("realloc failed\n");
        free(ptr);
    } else {
        printf("realloc succeeded\n");
        free(new_ptr);
    }
    return 0;
}
```

## Завдання 4.5 - Поведінка realloc() у випадку неможливості виділення пам’яті

Функція realloc() використовується для зміни розміру вже виділеної області пам’яті.

Прототип функції:
```
void *realloc(void *ptr, size_t size);
```
Параметр ptr є вказівником на раніше виділену область пам’яті, а параметр size визначає новий розмір цієї області.

Якщо realloc() може змінити розмір пам’яті, вона повертає вказівник на новий або той самий блок пам’яті.
Якщо ж система не може виділити необхідний обсяг пам’яті, функція повертає NULL.

При цьому важливо, що початковий блок пам’яті не звільняється, і його необхідно звільнити вручну за допомогою free().

## Код програми:
```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr = malloc(10 * sizeof(int));
    if (!ptr) {
        printf("Initial malloc failed\n");
        return 1;
    }
    printf("Initial allocation successful\n");
    size_t huge_size = (size_t)1024 * 1024 * 1024 * 1024;
    int *new_ptr = realloc(ptr, huge_size);
    if (!new_ptr) {
        printf("realloc failed\n");
        free(ptr);
    } else {
        printf("realloc succeeded\n");
        free(new_ptr);
    }
    return 0;
}
```

## Завдання 4.6 - Поведінка realloc() при використанні NULL та нульового розміру

Функція realloc() використовується для зміни розміру вже виділеної області пам’яті.

Прототип функції:
```
void *realloc(void *ptr, size_t size);
```
Функція realloc() має спеціальну поведінку у двох випадках.

Якщо ptr == NULL, виклик realloc(NULL, size) працює аналогічно до malloc(size), тобто виділяє нову область пам’яті.

Якщо size == 0, виклик realloc(ptr, 0) поводиться аналогічно до free(ptr). У цьому випадку раніше виділена пам’ять звільняється.

Для перевірки цієї поведінки було створено тестову програму.

## Код програми:
```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    printf("Case 1: realloc(NULL, size)\n");
    int *ptr = realloc(NULL, 5 * sizeof(int));
    if (ptr == NULL) {
        printf("realloc returned NULL\n");
    } else {
        printf("realloc worked like malloc\n");
    }
    printf("Case 2: realloc(ptr, 0)\n");
    void *result = realloc(ptr, 0);
    if (result == NULL) {
        printf("realloc(ptr, 0) returned NULL\n");
    } else {
        printf("realloc(ptr, 0) returned non-NULL pointer\n");
    }
    return 0;
}
```

## Завдання 4.7 - Використання reallocarray() замість realloc()

У даному завданні потрібно переписати код із використанням realloc() на варіант із використанням reallocarray().

### Початковий фрагмент коду:
```
struct sbar *ptr, *newptr;
ptr = calloc(1000, sizeof(struct sbar));
newptr = realloc(ptr, 500*sizeof(struct sbar));
```
Функція realloc() приймає вже обчислений розмір у байтах.
У випадку, якщо цей розмір обчислюється як добуток кількості елементів на розмір структури, існує ризик переповнення під час множення.

Функція reallocarray() призначена саме для таких випадків. Вона приймає кількість елементів та розмір одного елемента окремо, що робить код безпечнішим і зрозумілішим.

Переписаний варіант коду:
```
struct sbar *ptr, *newptr;
ptr = calloc(1000, sizeof(struct sbar));
newptr = reallocarray(ptr, 500, sizeof(struct sbar));
```
Для дослідження було створено дві тестові програми: одна з використанням realloc(), а друга - з використанням reallocarray().

## Код програми:
```
#include <stdio.h>
#include <stdlib.h>

struct sbar {
    int a;
    double b;
    char c;
};
int main() {
    struct sbar *ptr, *newptr;
    ptr = calloc(1000, sizeof(struct sbar));
    if (ptr == NULL) {
        printf("calloc failed\n");
        return 1;
    }
    newptr = realloc(ptr, 500 * sizeof(struct sbar));
    if (newptr == NULL) {
        printf("realloc failed\n");
        free(ptr);
        return 1;
    }
    printf("realloc succeeded\n");
    free(newptr);
    return 0;
}
```

## Завдання 4.8 - Дослідження lazy allocation у Linux

У цьому завданні досліджується механізм lazy allocation (ледачого виділення пам’яті) в операційній системі Linux.

Під час виклику malloc() операційна система не завжди одразу виділяє фізичну пам’ять. Спочатку резервується лише віртуальний адресний простір процесу. Фізична пам’ять виділяється лише тоді, коли програма починає використовувати ці сторінки.

Для аналізу використовуються параметри процесу:

VmSize — розмір віртуальної пам’яті процесу

VmRSS — обсяг фізичної пам’яті, який реально використовується

## Код програми:
```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

void print_mem() {
    FILE *f = fopen("/proc/self/status", "r");
    char line[256];

    while (fgets(line, sizeof(line), f)) {
        if (strncmp(line, "VmSize:", 7) == 0 ||
            strncmp(line, "VmRSS:", 6) == 0) {
            printf("%s", line);
        }
    }

    fclose(f);
}

int main() {

    size_t size = 1024L * 1024L * 1024L; 

    printf("Before malloc:\n");
    print_mem();

    char *ptr = malloc(size);

    printf("\nAfter malloc:\n");
    print_mem();

    for (size_t i = 0; i < size; i += 4096) {
        ptr[i] = 1;
    }

    printf("\nAfter touching memory:\n");
    print_mem();

    free(ptr);

    return 0;
}
```

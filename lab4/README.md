## Дослідження обмежень виділення пам'яті у Linux

У даній практичній роботі досліджуються обмеження виділення оперативної пам’яті у середовищі Linux за допомогою функції malloc() мови програмування C.

Особлива увага приділяється максимальному обсягу пам’яті, який може бути виділений за один виклик функції malloc(), а також впливу архітектури системи на це обмеження.

## Мета роботи 

Дослідити максимальний обсяг пам’яті, який може бути переданий функції malloc() за один виклик, та зрозуміти обмеження, що виникають через тип даних size_t і архітектуру операційної системи.

## Завдання 4.1 - Дослідження максимального обсягу пам’яті для malloc()

Функція malloc() використовується у мові програмування C для динамічного виділення пам’яті.

Прототип функції має вигляд:
```
void* malloc(size_t size);
```
Параметр size визначає кількість байтів пам’яті, яку потрібно виділити.

Тип даних size_t є беззнаковим цілим типом, який використовується для зберігання розміру об’єктів у пам’яті.

У 64-бітних системах Linux тип size_t має розмір 8 байтів, тобто 64 біти. Теоретично це дозволяє передати у функцію malloc() значення:

2^64 байтів

що приблизно дорівнює 16 ексабайтам пам’яті.

Однак у реальних умовах максимальний обсяг пам’яті, який може бути виділений за один виклик malloc(), становить приблизно:

2^63 байтів

тобто приблизно 8 ексабайт.

Це пояснюється тим, що частина бітів використовується внутрішніми механізмами управління пам’яттю, а також існують обмеження адресного простору процесу.

Для перевірки максимального значення типу size_t була створена тестова програма мовою C.

## Код програми:
```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    size_t max_size = (size_t)-1;
    printf("Maximum value of size_t: %zu\n", max_size);
    void *ptr = malloc(max_size);
    if (ptr == NULL) {
        printf("Memory allocation failed\n");
    } else {
        printf("Memory allocated\n");
        free(ptr);
    }
    return 0;
}
```

## Очікуваний результат

Під час виконання програми операційна система не зможе виділити настільки великий обсяг пам’яті, тому функція malloc() поверне NULL.

Це підтверджує наявність практичних обмежень на виділення пам’яті, навіть якщо теоретично тип size_t дозволяє передавати дуже великі значення.
